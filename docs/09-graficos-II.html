Gr√°ficos espaciales

Hay dos grandes tipos de datos espaciales: grillas regulares y grillas irregulares.

Grillas regulares

En la segunda secci√≥n de lectura de datos viste c√≥mo leer archivos NetCDF:

temperatura <- ReadNetCDF("datos/temperatura.nc", vars = "air",
                          subset = list(level = 1000,
                                        lat = c(-55, -20),
                                        lon = c(280, 310)))

La variable temperatura es una tabla que tiene dato de temperatura del aire para cada tiempo, nivel (altura), latitud y longitud. Como este archivo tiene informaci√≥n s√≥lo para un tiempo y el c√≥digo filtr√≥ s√≥lo datos de un nivel, entonces la variaci√≥n es s√≥lo en latitud y longitud. ¬øC√≥mo se grafica?

Se puede dibujar un punto por cada dato:

ggplot(temperatura, aes(lon, lat)) +
  geom_point()

Esto no dice nada, aunque podemos confirmar que los datos est√°n en una grilla regular. Para visualizar la temperatura, hay que mapear √©ste valor a un par√°metro est√©tico, por ejemplo, el color:

ggplot(temperatura, aes(lon, lat)) +
  geom_point(aes(color = air))

Puntos redondos y separados no es la forma m√°s informativa de ver estos datos. Una grilla regular en realidad define como una ‚Äúimagen‚Äù donde cada punto es un pixel. Esto se llama un ‚Äúraster‚Äù y con ggplot se puede graficar con geom_raster().

ggplot(temperatura, aes(lon, lat)) +
  geom_raster(aes(fill = air))

geom_raster() dibuja rect√°ngulos de igual ancho y alto en cada x, y. El ancho y el alto se definen a trav√©s de la resoluci√≥n de los datos. Funcionan perfectamente para grillas regulares. Estos rect√°ngulos tienen un color interno, por lo que el par√°metro est√©tico se llama ‚Äúfill‚Äù.

Desaf√≠o

ggplot2 tiene tres funciones para generar rect√°ngulos: geom_raster(), geom_tile() y geom_rect(). ¬øCu√°l es la diferencia entre cada uno? Fijate en la documentaci√≥n.

Mapas

Bien, hasta ah√≠ ten√©s un gr√°fico con los datos, pero para entenderlos en su contexto espacial como m√≠nimo hace falta un mapa. Para tener un mapa se necesitan datos de la localizaci√≥n de las costas o las fronteras. Existen varias fuentes de estos datos, pero una muy buena es Natural Earth. El paquete {rnaturalearth} provee una interfaz amigable para usar estos datos directamente en {ggplot2}.

Primero hay que cargar el mapa que queramos. Por ejemplo, para graficar el mapa de Argentina y sus pa√≠ses lim√≠trofes cargamos los datos con ne_countries():

mapa <- rnaturalearth::ne_countries(country = c("argentina", "chile", "uruguay", 
                                                "paraguay", "brazil", "bolivia", 
                                                "falkland islands"), 
                                    returnclass = "sf")

El argumento country es un vector con los pa√≠ses que necesitamos. El argumento returnclass es un poco t√©cnico, pero hace referencia a la estructura que queremos que devuelva. En este caso, returnclass = "sf" hace que devuelva un objeto de clase ‚ÄúSimple Features‚Äù. Las ‚ÄúSimple Features‚Äù son cualquier cosa menos simple internamente pero con {ggplot2} se pueden graficar con un geom espec√≠fico:

ggplot(mapa) +
  geom_sf()

Por defecto, el mapa se dibuja con un fondo gris, pero el problema es ese fondo va a tapar los datos! Paro para dibujar s√≥lo los contornos hay que modificarlo un poco:

ggplot(mapa) +
  geom_sf(fill = NA, color = "black", size = 0.2)

Cuando uno usa mucho un mapa, muchas veces termina siendo √∫til guardar el geom por separado en una variable.

mi_mapa <- geom_sf(data = mapa, inherit.aes = FALSE, fill = NA, color = "black", size = 0.2)

Adem√°s de las modificaciones est√©ticas, esta llamada a geom_sf tiene el argumento data = mapa, ya que no va a usar los datos ‚Äúglobales‚Äù de la llamada a ggplot(). Tambi√©n tiene √¨nherit.aes = FALSE porque tampoco tiene que ‚Äúheredar‚Äù los aes globales.

Entonces ahora, para graficar la temperatura y el mapa encima, armamos el gr√°fico de raster igual que antes y le sumamos el mapa:

ggplot(temperatura, aes(lon, lat)) +
  geom_raster(aes(fill = air)) +
  mi_mapa

Desaf√≠o

¬øPod√©s explicar qu√© sali√≥ mal?

Existen dos convenciones b√°sicas para medir la longitud, medida entre 0¬∫ y 360¬∫ o entre -180¬∫ y +180¬∫. Los datos de temperatura usan la primera y los del mapa, la segunda. Para convertir entre una y otra convenci√≥n, pod√©s usar la funci√≥n ConvertLongitude() de {metR}:

temperatura <- mutate(temperatura, lon = metR::ConvertLongitude(lon, from = 360))
ggplot(temperatura, aes(lon, lat)) +
  geom_raster(aes(fill = air)) +
  mi_mapa

Lo √∫nico que le falta a este gr√°fico es lo que le sobra: todo el espacio donde no hay datos. Para recortar el √°rea del gr√°fico y que tome s√≥lo donde tenemos datos de temperatura, hay que especificar los l√≠mites del sistema de coordenadas. Como mi_mapa es un geom_sf, hay que usar coord_sf.

ggplot(temperatura, aes(lon, lat)) +
  geom_raster(aes(fill = air)) +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

Otra forma de graficar datos espaciales es pensarlos como una superficie tridimensional donde el x y el y son las coordenadas y la altura (la coordenada z) es proporcional al valor de la variable. Imagin√°ndoselo as√≠, este gr√°fico de temperatura ser√≠a una monta√±a con su cima en Paraguay y su punto m√°s bajo cerca de Chubut.

Precisamente para esta situaci√≥n est√°n pensadas las l√≠neas de contorno que se pueden graficar en {ggplot2} usando la funci√≥n geom_contour().

ggplot(temperatura, aes(lon, lat)) +
  geom_contour(aes(z = air)) +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

Las l√≠neas dibujadas por geom_contour() son l√≠neas de nivel que unen puntos de valor constante. Sin embargo, esas l√≠neas azules no dan ninguna indicaci√≥n del valor de la variable por lo que es conveniente mapear el color de las l√≠neas ese valor constante.

ggplot(temperatura, aes(lon, lat)) +
  geom_contour(aes(z = air, color = stat(level))) +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

¬øDe d√≥nde sali√≥ ese stat(level)? Algunos geoms que realizan transformaciones estad√≠sticas computan variables nuevas que luego se pueden mapear a par√°metros geom√©tricos. ‚Äúlevel‚Äù es una de esas variables computadas de geom_contour. Para decirle a ggplot que al level al que te refer√≠s es a la variable computada, se rodea a la variable con la funci√≥n stat().

Una variedad de las l√≠neas de contorno son las l√≠neas ‚Äúllenas‚Äù. Hasta hace poco, {ggplot2} no ten√≠a una forma de graficar estos contornos llenos, as√≠ que est√° implementado en {metR} con la function geom_contour_fill(). Recientemente {ggplot2} implement√≥ la funci√≥n geom_contour_filled() pero a nosotros nos gusta m√°s la versi√≥n de {metR} üòâ

ggplot(temperatura, aes(lon, lat)) +
  metR::geom_contour_fill(aes(z = air)) +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

Por √∫ltimo, tranquilamente se pueden usar ambos geoms para resaltar los l√≠mites y

ggplot(temperatura, aes(lon, lat)) +
  metR::geom_contour_fill(aes(z = air)) +
  geom_contour(aes(z = air), color = "black") +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

Grillas irregulares

¬øQu√© pasa cuando en vez de observaciones organizadas en una bella grilla regular, ten√©s observaciones puntuales en lugares dispersos? Si te acord√°s de la secci√≥n de uniones, en la carpeta datos hay un archivo con datos de temperatura del servicio meteorol√≥gico y metadatos con sus ubicaciones. Esto define datos espaciales de temperatura.

# Para trabajar con menos datos, nos quedamos con la temperatura m√°xima media 
# de cada estaci√≥n.
observaciones <- readr::read_csv("datos/observaciones_smn.csv") %>% 
  group_by(station) %>% 
  summarise(tmax_media = mean(tmax, na.rm = TRUE))

estaciones <- read_csv("datos/estaciones_smn.csv") 

observaciones <- left_join(observaciones, estaciones, by = c("station" = "nombre")) %>% 
  filter(provincia != "ANTARTIDA")

Lo primero que se puede hacer es graficar estos datos de estaci√≥n con puntos, de la misma forma que para los datos regulares.

ggplot(observaciones, aes(lon, lat)) +
  geom_point(aes(color = tmax_media)) +
  mi_mapa +
  coord_sf(ylim = c(-55, -20), xlim = c(-80, -50))

Desaf√≠o

Grafic√° estos puntos usando geom_raster() y geom_contour(). ¬øCu√°l es el resultado?

Como estos datos no est√°n en una grilla regular, geom_raster() y geom_contour() no los pueden graficar. Para usar contornos hay que interpolar a una grilla regular. Una forma de hacerlo es usando la t√©cnica de kriging. Un paquete de R que la implementa es {kriging} en su funci√≥n kriging

Si vas a la ayuda de kriging(), vas a ver que requiere un vector de coordenadas x, un vector de coordenadas y y un vector de valores observados. Estos son las columnas lon, lat y tmax_media de la tabla observaciones. Entonces el c√≥digo ser√≠a:

observaciones_regular <- kriging::kriging(x = observaciones$lon, 
                                          y = observaciones$lat,
                                          response = observaciones$tmax_media)
## Error in onedim(response, n): NA/NaN/Inf in foreign function call (arg 1)

Para controlar el dominio en el cual se realiza la interpolaci√≥n hay que usar el argumento polygons de kriging(). Ah√≠, hay que poner puntos en x e y que definan el pol√≠gono dentro del cual se va a hacer la interpolaci√≥n. ¬øDe d√≥nde sac√°s un pol√≠gono con el contorno de Argentina? ü§î. ¬°Lo est√°s mirando! Es nuestro mapa.

La funci√≥n fortify() de {ggplot2} genera pol√≠gonos a partir de la salida de {rnaturalearth}. De esos pol√≠gonos hay que seleccionar √∫nicamente las coordenadas de longitud y latitud, y al mismo tiempo, renombrarlas como x e y para que kriging() las entienda. Finalmente, hay que meter todo eso en una lista.

poligonos_arg <- rnaturalearth::ne_countries(country = "argentina") %>% 
  fortify() %>% 
  select(x = long, y = lat) %>% 
  list()

Ahora, hay que repetir todo lo anterior, pero usando con polygons = poligonos_arg.

observaciones_regular <- observaciones %>% 
  na.omit() %>% 
  with(kriging::kriging(lon, lat, response = tmax_media, polygons = poligonos_arg)) 

observaciones_regular <- observaciones_regular$map %>% 
  rename(lon = x, lat = y, tmax_media = pred)

ggplot(observaciones_regular, aes(lon, lat)) +
  geom_contour_fill(aes(z = tmax_media)) +
